# League Automation Center

The League Automation Center is the event-driven communication layer for the Lobo Infinity League Portal.

Discord is the first active destination, but automation is designed around league events and destination rules so future email, push, public API, and other delivery channels can be added without changing league business logic.

## Event Architecture

Every automation item begins as a League Event.

Each event contains:

- Event ID
- Event Type
- Category
- Priority
- Player
- Division
- Timestamp
- Message
- Payload
- Destination rules
- Queue status

Supported event types include:

- Game Submitted
- Game Corrected
- Achievement Unlocked
- Promotion
- Relegation
- Season Reminder
- Season Started
- Season Ended
- Division Champion
- League Champion
- Hall of Fame
- Record Broken
- Army List Submitted
- Army List Approved
- Player Joined
- Identity Linked
- Commissioner News
- Stream Scheduled
- System Warning
- Integrity Alert

## Destinations

Each event type can independently enable or disable:

- Portal
- Discord
- Timeline
- News
- Commissioner Feed
- Email
- Push
- Public API

Discord is currently the active external delivery destination. Other destinations are queued and recorded so the event architecture is ready for future delivery workers.

## Automation Rules

Automation rules are stored in the `Automation Rules` sheet.

Rules define:

- Whether the event is enabled.
- Which destinations receive the event.
- Priority.
- Template ID.

Commissioners manage these rules from the League Automation Center.

## Templates

Templates are stored in the `Automation Templates` sheet.

Each template contains:

- Template ID
- Event Type
- Name
- Title
- Body
- Discord Format
- Enabled

Templates support placeholders such as `{{player}}`, `{{division}}`, and `{{message}}`. The first implementation stores and edits templates; Discord delivery uses the event payload and manual embed formatting while preserving the template architecture for future expansion.

## Queue

Every enabled destination creates a queue item in the `Automation Queue` sheet.

Queue items track:

- Queue ID
- Event ID
- Event Type
- Destination
- Status
- Timestamp
- Attempts
- Last Attempt
- Reason
- Payload

Statuses include:

- Pending
- Sent
- Retry
- Failed

Failures never interrupt league operations. The original league action continues, and the Automation Center records the failed queue item for retry.

## Discord Setup

1. Create a Discord webhook for the target league channel.
2. Open the League Automation Center as a Commissioner.
3. Paste the webhook URL into Discord Destination settings.
4. Set the announcement channel label.
5. Enable Discord automation.
6. Click Test Connection.
7. Confirm a test message appears in Discord.

Webhook URLs are stored in the backend `Discord Config` sheet and are never returned to public frontend payloads. The UI only shows a masked webhook value.

## Deep Links

Automation events should include permanent portal links generated by `DeepLinkApi.gs`.

Discord is the first destination using these links. Each embed should route directly to the relevant page:

- Game result: `/game/{gameId}`
- Achievement: `/achievement/{achievementId}`
- Player event: `/player/{leaguePlayer}`
- Promotion or relegation: `/career/{leaguePlayer}`
- Hall of Fame: `/hall-of-fame`
- Weekly recap: `/weekly-report`
- News: `/news/{newsId}`
- Stream: `/stream/{streamId}`
- Army list: `/army-list/{listId}`

Do not manually build portal URLs inside destination services.

## Commissioner Controls

The Automation Center includes:

- Run Automation
- Pause Automation
- Resume Automation
- Replay Last Event
- Replay Week
- Replay Season
- Clear Queue
- Retry Failed
- Send Now

Assistant Commissioners can view automation and use announcement controls where existing permissions allow it. Commissioners manage configuration and queue cleanup.

## Weekly Automation

Weekly automation publishes recap events for:

- Weekly standings
- Player of the week
- Achievement review
- Promotion race
- Relegation race
- Upcoming games
- League health
- Commissioner summary

The initial implementation records these as league events and queues configured destinations.

## Troubleshooting

If Discord messages do not appear:

1. Confirm automation is running.
2. Confirm Discord is connected.
3. Confirm the event type has Discord enabled.
4. Confirm the webhook URL is valid.
5. Check the Queue panel for Retry or Failed items.
6. Check the Discord Automation Log sheet for HTTP response details.
7. Click Retry Failed after fixing configuration.

Webhook failures must never block game submissions, achievements, news, streams, army lists, or commissioner operations.
